REVIEW NOTES FROM FULL-STACK-102

What is React?

React is a JavaScript library for building UIs (User Interfaces). It's one of the most popular JavaScript front-end frameworks used today. It was released by Facebook in 2013, and still used today for many applications. React continues to be popular for its component-based architecture which allows programmers to reuse code, minimal number of updates needed to make in the DOM, a large user community allowing formany resources, and its flexibility to be used in many devices.

Benefits of using React:

The create-react-app is a very beginner friendly tool that allows you to set up a react repo instantly. All you need a VirtualDOM and JSX.

  • Efficient debugging
  • Ability to reuse code components
  • Accessible through JavaScript library
  • Use of VirtualDOM
  • Great for Search Engine Optimization
  • Beginner friendly
  • Works on iOS and Android apps
  • Large user community
  • Many tools for experimenting
  • Worldwide usage across prominent brands

What is AWS?

Amazon Web Services is the world's most comprehensive and boradly adopted cloud platform. A cloud platform refers to the operating system and hardware of a server in a internet-based data center. It contains cloud storage, which means that remoter servers are used to save data in a virtual machine in a physical server. AWS offers over 200 fully featured services from data centers, globally! Companies are turning to AWS for various reasons, such as more functionality, larger community, more secure, more innovative, and most proven expertice.

Why is it important?

Because most businesses are switching over to AWS, it has become a crucial part of the tech industry. More and more companies are seeking employees with AWS skills nowadays and having these skills allow for a higher pay career as well. Luckily, AWS offers a free tier for the first 12 months of creating an account, which allows people to gain free, hands-on experience. AWS also has an official training and certification page as a resources to get more in-depth knowledge about AWS, which can carry over to the job. Employers are strongly suggesting people to consider those certifications as it will make your application more attractive.

Which role at Ally uses it?

In addition to the AIPD (Auto Insurance and Product Delivery), the IPRM (Information Protection and Risk Management) role at Ally also uses AWS, as it is a cybersecurity role. There are various roles in this rotational department. The job itself consists of employees participating in ecurity inversigations, collaborate in critical IT projects, oversee and assist in supporting technologies, oversee and define testing criteria for systems and applications, coordinating scanning policies and schedules with business and operating teams, and analyze the results of audits. I imagine that one of Ally's uses of AWS is for their cybersecurity services, which consist of high quality protection of data, managing identities, the enforcement of fine-grained security policy, and continuous monitoring of network activities and account behaviors within the cloud environment.

CYBERSECURITY FUNDAMENTALS

Per Ally, it was suggested that employees who will be starting the rotational to have a foundational knowledge and principles of cybersecurity, such as understanding threats, vulnerabilities, and basic security concepts.

Overview

  • CIA (Confidentiality, Integrity, and Availability) Triad:

    • Confidentiality, Integrity, and Availability are the core principles of information security. Confidentiality ensures that data is protected from unauthorized access, integrity ensures that data is accurate and unaltered, and availability ensures data is accessable when needed.

    Readings:
    • FortiNet
    • UniTrends
    • SecurityScorecard
  • BCDR (Business Continuity and Disaster Recovery) Principles:

    • BCDR refers to the processes and procedures put into place to ensure the continuation of business operations in the event of a disaster or disruptive event. It consists of having a backup of data, systems, and infrastructures.

    Readings:
    • Datto
    • TechTarget
    • Axcient
    How the pandemic has affected BCDR:
    • UCToday
    • ComputerWeekly
  • Risk Management:

    • Risk management involves identifying, assessing, and prioritizing risks to an organization's assets, including information and systems.

    Readings:
    • HyperProof
    • Imperva
    • Global Risk Management Institute
    • BusinessTech
  • IAM (Identity and Access Management):

    • IAM refers to the processes and technologies used to manage and control systems, applications, and data. It involves user authentication authorization, and identity lifecycle management.

    Readings:
    • FortiNet
    • OneLogin
    • Emeritus
    • CSO
  • Computer Networking:

    • Computer networking invloves design, implementation, and management of networks that eases communication and data transfer between devices and systems. While computer networking is not completely focused on security, it is still important to understand computer networking to implement network security measures.

    Readings:
    • GeeksForGeeks
    • AWS
    • Indeed
  • Network Security:

    • Network security focuses on protecting the integrity, confidentiality, an availability of network resources and data. It also includes measures such as firewalls, intrusion detection systems, and virtual private networks (VPNs).

    Readings:
    • Herzing University
    • TechTarget
    • CISCO
  • Data Security:

    • Data security involves protecting data from unauthorized access, disclosure, alteration, or destruction. It consists of measures like encryption, access control, and data backup.

    Readings:
    • FortiNet
    • Varonis
    • IntelliPaat
    • CFI
  • Cloud Security:

    • Cloud security refers t the protection of data, applications, and infrastructure in cloud computing environments. It involves securing cloud-based services, data privacy, and compliance with regulations.

    Readings:
    • Box
    • Citrix
    • CheckPoint
  • System Hardening Principles:

    • System hardening refers to the process of configuring and securing computer systems to reduce their vulnerabilities and make them more resistent to attacks. It includes activities like removing unnecessary software, applying secure patches, and configuring secure settings.

    Readings:
    • BeyondTrust
    • Netwrix
