Learning From Udemy

This week I have been focusing on the learning and implementation of the programming language, Python. I took a Udemy Course to learn the basics of the language. Luckily it was easier than I thought to pick up on the basics. The logic of the language is similar to Java and JavaScript. They syntax is of course, very different. I had to keep fighting the urge to end a statement with a semi-colon. I was surprised to see how strict the indenting was for the code to run. Normally I can use a keyboard short-cut to help format the code and it can usually tell me if I am missing something such as a semi-colon; this was different in Python. I kept running into errors or a piece of code would not run correctly, only to find out some time later the indenting was off.

As you can see in the carousel above, I followed along with all the coding in the Udemy course I took. The arithmetic runs the same as we have learned with operations. With this I was also able to create a calculator (below). Also, what we know as objects, which remain objects as data collections, are called dictionaries in Python. They contain a key/value pair, which was what I used to create a quiz about Python basics (below as well). One of the pieces of code that I spent way too much time on trying to figure out why the code was not running as it was supposed to was the factorial. I realized about an hour after staring at it, was not working because the return statement was nested in the for-loop scope.

Similar to the programming languages we have learned, Python is also an object oriented programming language. You will notice that I made a 'Person' class that was instantiated with '__init__'. You will also notice the getName() and getAge() methods that can be called when an object is created to set a name and age. Python also allows for class inheritance. This is evident in the SportsCar class where it inherited the Car class and overrode the methods to match its personal characteristics.
