NETWORK INFRASTRUCTURE BASICS

"Network infrastructure refers to the hardware and software that enable network connectivity and communication between users, devices, apps, the internet, and more. "
- CISCO
  • IP Addresses (Internet Protocol Address): A unique string of characters that identifies each device and can be used for devices to communicate with each other.
  • Subnet mask: A 32-bit value used with IP addresses to divide a network into subnetworks(subnets). It helps identify which part of the IP address represents the network and which part represents the host.
  • Default gateway: The IP address of the router or gateway device that connects a LAN to external networks.
  • DNS (Domain Name System): A system that translates domain names into IP addresses. It allows users to visit websites and other network resources using human-readable domain names.
  • LAN (Local Area Network): A network that connects devices within a limited area, like a home, office, or school. It allows the devices to share resources, like printers and internet access.
  • WAN (Wide Area Network): A network that covers a large geographical area, usually connecting multiple LANs together. It allows for long-distance communication and connection.
  • Switch: A networking device that connects multiple devices within a network and eases communication between them. It operates in the data link layer of the network protocol and forwards data packets to their intended destination based on MAC addresses.
  • Router: A networking device that connects multiple networks together and routes data packets between them. It operates in the network layer of the network protocol and uses IP addresses to determine the most efficient path to transmit data.
  • Bandwidth: Max amount of data that can be transmitted over a network connecting withing a given period. It is usually measured in bits per second (bps) and represents the capacity of the network connection.
  • Latency: Amount of time it takes for the data packets to travel from the source to the destination across a network. Often measured in milliseconds (ms) and represents the delay or responsiveness of the network connection.


PROGRAMMING LANGUAGES

A few used at Ally

Based on the information received from Ally through CTAC, I realized that the more common programming languages in the rotational roles we will be going into, are:

  1. Python
    • Programming language known for it's simplicity and readability
    • Can be used for wed development, scientific computing, data analysis, machine learning, and automation
    • Most common for back-end development, but can be used with front-end
    • Due to an extensive standard library and many libraries, developers use it build apps quickly and efficient
  2. Bash (Bourne Again SHell)
    • Unix shell and command language
    • Mostly used in command-lines and scripting tasks in Unix and Linux systems
    • Often used for implementing repetitive tasks, writing shells, and executing administration tasks
    • Although it is not typically used for front-end or back-end web development, it is a useful tool for system administrators and developers in Unix-like environments
  3. ReactJS
    • A JavaScript library used for building User Interfaces
    • Mostly used with front-end web development and suited for creating single-page applications (SPA's)
    • Follows a component based architecture, which allows developers to build reusable UI components and efficiently manipulate applications
    • Provides a declarative and efficient way to create interactive and dynamic UIs, resulting in a better user experience
  4. Spring Boot JPA (Java Persistence API)
    • A Java-based framework used for building stand-alone, production-grade applications
    • JPA has the ability to access relational databases from Java applications
    • Spring Boot and JPA provide a useful framework for developing back-end applications with Java
    • Spring Book simplifies the configuration and deployment of Java applications, while JPA provides an abstraction layer to interact with databases
    • The combination of these two allows developers to build scalable, efficient, and maintainable back-end systems that handle data
  5. Angular +2
    • A TypeScript-based framework for building web applications
    • Mostly used for front-end development and focuses on creating responsive single-page applications
    • Also follows a component based architecture, similar to React, allowing developers to build reusable UI components
    • Provides data binding, routing, and testing utilities
    • Provides a structures and scalable approach to building complex web applications, with a strong community support and plenty of tools
  6. R
    • A programming language and software environment commonly used for statistical computing, data analysis, and visualization
    • Provides a wide range of packages and libraries for statistical modeling, machine learning, and data manipulation
    • Powerful statistical capabilities and a plethora of packages for many data related tasks

Other Tools

  • Ansible: open-source automation tool used for configuration management, application deployment, and task automation. It allows you to define and manage infrastructure as code, simplifying network and applications.
  • Terraform: open-source infrastructure as code tool. With Terraform you can describe your desired infrastructure configuration using domain-specific language or JSON, then Terraform handles the provisioning and management of the resources. It supports a rang of cloud providers, including AWS, Azure, Google Cloud Platform, and more.
  • Palo Alto: Network cybersecurity company that provides a range of network security products and services. Their products include firewalls, threat detection systems, secure access solutions, and cloud security offerings. Their firewalls are known for their advanced features and capabilities in securing networks, and protecting against various types of cyber threats.
  • Splunk: A software platform used for searching, analyzing, and visualizing machine generates data such as logs, events and metics. It collects indexes and data from various sources, allowing organizations to gain insight, troubleshoot issues, detect anomolies, and monitor system performances. It is most commonly used for log management, security information and event management (SIEM), and operational intelligence.
  • Digital Guardian: A data protection platform that helps organizations secure sensitive data and prevent data breaches. It offers capabilities such as data loss prevention, data visibility and control, user activity monitoring, and threat detection. Digital Guardian also helps protect data both on-premises and in the cloud, ensuring complience with data privacy regulations and mitigating the risks of data leakage.
  • Linux: An open-source operating system used for a wide range of devices, from servers and desktop computers to embedded systems and smartphones. It provides a strong and secure platform for running applications and managing resources. Linus is known for it's stability, flexibility, and extensive community support. It is used widely in web servers, cloud infrastructure, and many other computing environments.
  • AWS (Amazon Web Services): A cloud computing platform offered by Amazon. It provides a wide range of cloud services including compute power, storage, databases, networking,security, and machine learning.
  • Crowd Strike: A cybersecurity company that specializes in endpoint protection and threat intelligence. Falcon, their most recognized product is an endpoint protection platform that uses machine learning and behavioral analysis to detect and respond to cyber threats.
  • Microsoft Services: A wide range of services offered by Microsoft to support their services.
    • Excel: Widely used spreadsheet application for organizing, analyzing, and visualizing data in a tubular format.
    • Power Apps: Low-code development platform that allows users to build custom business applications withouth extensive coding knowledge.
  • Git: Distributed version control system used for tracking changes in source code during software development. It provides features for collaborative development and thus used to improve collaboration. It is also used to enable efficient code review processes and provide a reliable history of changes made in the code space.
  • SQL Servicers: Usually referred to as database servers, are software systems used for storing, managing, and retrieving structured data.
